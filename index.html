Sesi 2 · Praktek Langsung — 02

Yuk Bikin Skill: Kenalan Sama SKILL.md

Satu folder, satu skill. Ini struktur dasarnya.

nama-skill-kamu/
├── SKILL.md ← wajib, pintu masuk ├── references/ ← detail tambahan, dibaca pas perlu │ └── panduan-lengkap.md ├── scripts/ ← automasi yang deterministik │ └── generate.py └── assets/ ← starter file & output template └── template.pptx
📄 SKILL.md Wajib ada. Isinya: kapan skill ini dipakai (trigger) + langkah kerjanya. Ringkas. Detail dilempar ke file lain.
📚 references/ Dokumen pendukung yang lebih detail. Cuma dibaca AI kalau emang perlu, biar SKILL.md-nya gak kepanjangan.
⚙️ scripts/ Script (Python/bash/JS) buat bagian yang deterministik & berulang, biar AI tinggal jalanin dan gak "ngarang".
📁 assets/ Template, starter file, atau contoh output (docx/pptx/xlsx) yang AI tinggal isi & modifikasi.
💡

SKILL.md itu ringkas & to-the-point. Detail teknis, contoh, dan template taruh di folder pendukung. AI baca itu cuma pas dibutuhin (progressive disclosure).

Sesi 2 · Praktek Langsung — 03

Bedah Isi SKILL.md

Folder-nya udah kelar dibahas. Sekarang isi filenya, mulai dari description.

prd-writer/SKILL.md
--- name: prd-writer description: Bikin draft PRD terstruktur dari brief singkat. Dipakai kalau user minta "buatin PRD" atau kasih ide fitur mentah yang perlu didokumentasiin. --- # PRD Writer ## Langkah Kerja 1. Tanya konteks: masalah apa, buat siapa 2. Susun draft: Problem, Goal, Solution, Metrics 3. Kasih ke user buat direview
🏷️ name Nama teknis skill-nya. Huruf kecil, pakai tanda hubung, harus sama persis kayak nama foldernya.
📡 description Paling penting. Jelasin apa yang dikerjain plus kapan harus dipanggil, kata kunci atau situasi pemicunya.
🧭 Judul & Heading Judul singkat setelah frontmatter, biar orang (dan Claude) langsung ngerti ini skill apa.
📝 Langkah Kerja Instruksi konkret, step-by-step. Ini yang beneran diikutin Claude pas eksekusi.
💡

Bingung mulai dari mana? Mulai dari description. Itu yang paling nentuin skill kamu bakal "ke-denger" sama Claude atau nggak.

Sesi 2 · Praktek Langsung — 04

Cara Install Skill-nya

Bedanya cuma di "tempat naruh". Caranya sama-sama gampang.

💬
Web / App Claude Chat
1Buka Settings → Capabilities, aktifin "Code execution and file creation"
2Masuk ke Settings → Customize → Skills
3Upload skill kamu (zip foldernya, bukan cuma file SKILL.md), atau tinggal toggle-on skill bawaan
4Claude otomatis baca SKILL.md & nampilin nama skill-nya
⌨️
Terminal Claude Code
1Bikin folder .claude/skills/nama-skill/ (khusus project ini) atau ~/.claude/skills/nama-skill/ (global)
2Taruh file SKILL.md kamu di dalamnya (plus references/scripts/assets kalau ada)
3Restart Claude Code
4Jalanin /skills buat mastiin skill-nya ke-load
💡

Claude Chat: upload lewat Settings. Claude Code: tinggal taruh file di folder. Sesederhana itu.

Sesi 2 · Praktek Langsung — 07

Bikin Workflow: Modular vs Linear

Dua cara nyusun kerja bareng multi-agent. Pilihannya tergantung kebutuhan.

🎯Modular

Ada 1 orchestrator yang mutusin sendiri agent mana yang perlu dipanggil, sesuai kebutuhan saat itu.

Orchestrator
⇢ Agent A ⇢ Agent B ⇢ Agent C
Plus / Minus
  • Fleksibel: orchestrator bisa adaptasi urutan sesuai kasus
  • Gampang di-scale, tinggal nambah agent baru
  • Hasil & urutan eksekusi bisa beda tiap run (kurang predictable)
  • Lebih mahal & lambat: ada overhead "mikir siapa yang dipanggil"
📍Linear

Kita yang tentuin flow-nya di depan. Abis A, lanjut B, lanjut C, dst. Urutannya fix.

Ambil Data → Analisa → Bikin Output → Cek Kualitas
Plus / Minus
  • Predictable & konsisten: gampang di-QA
  • Lebih murah & cepat, tanpa overhead delegasi
  • Kaku: kasus di luar flow yang udah ditentuin bisa nyangkut
  • Kurang scalable buat kasus yang variasinya banyak
💡

Gak ada yang paling benar. Modular buat kasus yang variatif & butuh judgment; Linear buat proses yang udah jelas & butuh konsistensi.

Sesi 2 · Praktek Langsung — 08

Gabungin Skill: Bikin Pipeline

Bikin satu skill baru yang tugasnya cuma manggil skill-skill lain, sesuai urutan.

User cuma manggil satu skill: /prd-to-uat-pipeline
prd-to-uat-pipeline/SKILL.md
Step 1: panggil skill-prd → hasil: PRD.md Step 2: feed PRD.md ke skill-uat-scenario → hasil: UAT-scenarios.md Step 3: gabungin dua-duanya jadi satu paket output
skill-prd → skill-uat-scenario → Output Gabungan
💡

User cuma perlu manggil satu skill. Urutan & "siapa manggil siapa" itu udah didokumentasikan di dalam pipeline-nya sendiri.

Sesi 2 · Praktek Langsung — 09

Nambah GTM & Legal? Ini Titik Baliknya

Gampang nambahin step baru ke pipeline. Tapi hati-hati sama sinyal ini.

Orchestrator
PRD → UAT (selalu) GTM? (kalau perlu) Legal Assessment? (kalau nyentuh compliance)
💡

Begitu ada skill yang sifatnya kondisional (gak selalu perlu jalan), itu sinyal buat pindah dari Linear ke Modular. Biasanya jadi hybrid: orchestrator yang mutusin skill kondisional mana yang jalan, sementara bagian yang emang selalu berurutan (PRD→UAT) tetap jalan sebagai satu pipeline kecil di dalamnya.
